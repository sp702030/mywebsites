About — Smart Parking System

Smart Parking System is a lightweight demo to manage parking slots for campuses and small parking areas. It focuses on quick booking, a simple PIN-based checkout, demo payments, and admin exports — all running inside the browser (localStorage) for easy testing and demonstration.

How it works — Overview

  • Booking: User selects a slot, provides name, vehicle number, phone and a 4-digit PIN. Booking timestamp is saved.
  • Payment (Demo): The app shows a demo QR modal — pressing Fake Pay marks booking as paid and stores a demo payment ID.
  • Checkout: When vehicle leaves, user/admin initiates checkout — the app asks for the 4-digit PIN. On success a receipt is generated and stored.
  • Receipts & Admin: Each checkout produces a receipt saved to localStorage. From Admin you can export bookings & receipts as CSV for backup or analysis.

Components

  • Parking page: Layout of slots, booking form, payment modal and booking list.
  • Admin page: Login-protected area (demo creds), analytics, bookings table and CSV export.
  • Home page: Overview, today's revenue (calculated from receipts) and quick links.

Security & Limitations

  • This is a demo using browser localStorage — not suitable for production without a backend.
  • Authentication is demo-only (username/password in client). For production move to server-side auth and HTTPS.
  • Payments here are simulated. Integrate a real payment gateway for real transactions.

Technical Notes

Receipts and bookings are JSON saved in localStorage keys: rcp (receipts) and book (bookings). CSV export converts these for offline storage.

Further improvements

  • Move storage to server (API + DB).
  • Secure admin with JWT / sessions.
  • Integrate real payment gateway (Razorpay / Stripe).
  • Add user accounts and notifications (SMS/Email).
